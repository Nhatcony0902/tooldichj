# Refresh Orchestrator

Specification for `scripts/refresh-orchestrator.cjs` — the engine behind both
`t1k diagram refresh` (CLI) and `/t1k:preview --refresh` (skill mode).

For the `.t1k-diagram-meta.json` schema, see §"Meta File Schema" below.
For the adapter contract (detect/generate/capabilities), see `adapter-contract.md`.

---

## Inputs

| Input | Default |
|-------|---------|
| `--out-dir <path>` | `docs/diagrams/` relative to cwd |
| `--engine <name>` | auto (run discovery) |
| `--types <csv>` | `modules,classes,packages,c4` |
| `--timeout <sec>` | 30 |

---

## Algorithm

### Step 1 — Resolve output directory and sandbox root

```
outDir = path.resolve(cwd, flagOutDir ?? 'docs/diagrams')
if !exists(outDir): fs.mkdirSync(outDir, { recursive: true })
sandboxRoot = fs.realpathSync(outDir)   // resolves symlinks AFTER creation
```

Create directory before `realpathSync` — deterministic on macOS where `/tmp → /private/tmp`.

### Step 2 — Acquire cross-command lock

Lock at `~/.claude/.diagram.lock` via `process-lock.ts`. Mutating commands (`install`, `refresh`,
`list --verify`) must hold this lock. Stale lock recovery: if holding PID no longer exists
(ESRCH on `process.kill(pid, 0)`), delete and re-acquire. Timeout: 10 minutes.

### Step 3 — Discover adapters

If `--engine <name>`: skip discovery, load that adapter directly.
Otherwise: run `adapter-discovery.cjs`. If no adapters found: print advisory to stderr and exit 0.

### Step 4 — Polyglot monorepo notice (H2)

When discovery returns multiple engine matches: run only the highest-priority adapter,
then print to stderr for each non-selected engine:
```
[t1k] Multiple adapters matched. Running: <selected>
[t1k] To run <other>: t1k diagram refresh --engine <other> --out-dir docs/diagrams-<other>
```
Multi-adapter fan-out (`--all`) is deferred to v2. Zero silent data loss.

### Step 5 — For each requested capability type

**5a. Pick adapter.** If no adapter supports this type: write `{type}.md` with
`⚠️ Not available for this engine — see README.md for details`. Record as `capabilities_skipped`.

**5b. SHA-256 protection check (BLOCKER 5).**
If the target file exists and its hash does not match the stored hash in `.t1k-diagram-meta.json`:
user edited it. Prompt interactively, or (non-interactive) write `<relPath>.new` sidecar, log warning, exit 0.
No "DO NOT EDIT" banner — protection comes from the hash, not the banner.

**5c. Pre-validate target path (fast fail).**
```
resolved = path.resolve(sandboxRoot, `${type}.md`)
assert resolved.startsWith(sandboxRoot + path.sep)
```

**5d. Invoke `generate.cjs` in refresh mode.**
```
execFileSync('node', ['scripts/generate.cjs', '--type', type, '--out-dir', sandboxRoot], {
  cwd: adapterSkillDir, timeout: timeoutMs, windowsHide: true,
  stdio: ['ignore', 'pipe', 'pipe']
})
```
Parse stdout as JSON `{ file, warnings, capabilities_skipped }`.
On failure: log error, mark capability failed, continue — never abort entire refresh.

**5e. Sandbox check on returned path (symlink-safe — H5).**
```
resolvedFile = path.resolve(sandboxRoot, result.file)
// If parent dir exists: assert fs.realpathSync(parentDir).startsWith(sandboxRoot + sep)
// If target exists: assert fs.realpathSync(resolvedFile).startsWith(sandboxRoot + sep)
// If target is a symlink: throw EACCES — T1K refuses to write through symlinks
```

**5f. Update meta file after write.**
```
meta.generated[result.file] = { sha256: sha256(content), generatedAt: new Date().toISOString() }
```

### Step 6 — Write outputs

Write `README.md` index (timestamp, git SHA, capability table with per-row status and adapter).
Write `.t1k-diagram-meta.json` atomically (temp file + rename).
Each generated diagram file starts with ONE informational comment line (not a big banner):
```
<!-- Generated by t1k diagram refresh at <timestamp> from adapter <name>@<version>. Hand-edits trigger a .new sidecar on the next refresh — see references/refresh-output-convention.md. -->
```

## Failure Modes

| Failure | Behaviour |
|---------|-----------|
| `detect.cjs` exits 1 | Adapter skipped — not an error |
| `generate.cjs` times out | Capability marked failed; continue to next type |
| Returned path outside `outDir` | Hard reject — path traversal attempt |
| Symlink target in `outDir` | Hard reject — EACCES |
| Hash mismatch (hand-edit) | Prompt (interactive) or write `<relPath>.new` sidecar, log warning, exit 0 |
| Lock held > timeout | Hard fail with pid + hint to delete lock |
| All capabilities skipped | Exit 0, print advisory |
| `out-dir` creation fails | Hard fail immediately with OS error |

## Meta File Schema

`.t1k-diagram-meta.json` is written atomically (temp file + rename) to `<out-dir>/.t1k-diagram-meta.json`.

```jsonc
{
  "timestamp": "<ISO8601>",           // refresh start time
  "git_sha": "<hex or null>",         // git rev-parse HEAD; null if not in a git repo
  "adapters": ["<adapter-name>@<ver>"],
  "out_dir": "<absPath>",
  "t1k_preview_version": "<semver>",
  "generated": {
    "<relPath>": {
      "sha256": "<hex>",              // SHA-256 of the file content written
      "generatedAt": "<ISO8601>"
    }
  }
}
```

The `generated` map covers every file written by the refresh, including `README.md` and all `{type}.md` files
(even capability-skipped ones). CI gate `validate-diagram-meta-schema.cjs` (T8d) schema-checks this file.
